Features Reference

ForceElementSizeFeature#

ForceElementSizeFeature

This feature sets element size programmatically and only once, when DOMContentLoaded has been fired.

ForceResponsiveViewportFeature#

ForceResponsiveViewportFeature

This feature inserts or replace a meta element looking like:

<meta name="viewport" content="width=device-width" />

HandleElementCSSBoxFeature#

~ API Reference ~
HandleElementCSSBoxFeature

This feature enables receiving the CSS Box dimensions of an element in the WebView pixels unit. The first element matching the provided tagName is retained. A new event will be triggered on every resize.

If you are looking for the document content size, use HandleHTMLDimensionsFeature instead.

HandleHashChangeFeature#

~ API Reference ~
HandleHashChangeFeature

This feature allows to intercept hashchange events, when the hash fragment of the URL changes. This could happen when the user clicks on anchors like this:

<a href="#subresource">Let's Jump to Subresource</a>

Or when JavaScript code imperatively changes the hash fragment of current location. See MDN‚Äîhashchange event.

HandleHTMLDimensionsFeature#

~ API Reference ~
HandleHTMLDimensionsFeature

This feature enables receiving various dimensions relative to the layout. The events will only be fired when a change is observed to either the layout or the content size.

If you need to guarantee that 1 CSS pixel = 1 Device pixel, you should use this feature with a meta tag viewport setting width to device width. See ForceResponsiveViewportFeature.

HandleLinkPressFeature#

~ API Reference ~
HandleLinkPressFeature

This feature allows to intercept clicks on anchors such as:

<a href="https://domain.com/">Let's Travel</a>

By default, it will prevent the click from propagating. But you can disable this option.

If you need to intercept click on hash fragments, consider HandleHashChangeFeature instead.

HandleVisualViewportFeature#

~ API Reference ~
HandleVisualViewportFeature

This feature adds a handler triggered when the visual viewport changes. It requires VisualViewport API support on browsers (iOS Safari 13 and Android WebView 62). See MDN‚ÄîVisualViewport API.
